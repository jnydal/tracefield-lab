Plan: Surface computed feature availability in the analysis job form
Context

Analysis job creation uses GET /features/definitions (the schema registry) to populate feature dropdowns. No endpoint exists to query what is actually computed in the features table. When a researcher selects total_incidents and units_sold_thousands but scalar extract was never run (or ran for different entities), the job either fails or returns ρ ≈ 0 with no actionable error. This is IMPROVEMENTS.md §12.

The fix has two parts:

A new GET /features/summary API endpoint that LEFT JOINs feature_definitions with features and returns each definition with its computed row count.
The analysis job form uses the summary instead of the bare definitions list — option labels show the count, and features with zero rows get an inline amber warning when selected.

Step 2 — Application.kt: add GET /features/summary
Add after the closing brace of route("/features/definitions") { … } (around line 1451):

get("/features/summary") {
    val items = transaction(DatabaseManager.getDatabase()) {
        FeatureDefinitions
            .join(Features, JoinType.LEFT, FeatureDefinitions.id, Features.featureDefinitionId)
            .slice(
                FeatureDefinitions.id,
                FeatureDefinitions.name,
                FeatureDefinitions.description,
                FeatureDefinitions.valueType,
                FeatureDefinitions.unit,
                Features.id.count()
            )
            .selectAll()
            .groupBy(
                FeatureDefinitions.id,
                FeatureDefinitions.name,
                FeatureDefinitions.description,
                FeatureDefinitions.valueType,
                FeatureDefinitions.unit
            )
            .orderBy(FeatureDefinitions.name)
            .map { row ->
                FeatureSummaryResponse(
                    id = row[FeatureDefinitions.id].value.toString(),
                    name = row[FeatureDefinitions.name],
                    description = row[FeatureDefinitions.description],
                    valueType = row[FeatureDefinitions.valueType],
                    unit = row[FeatureDefinitions.unit],
                    computedCount = row[Features.id.count()].toInt()
                )
            }
    }
    call.respond(items)
}
Features and JoinType are already available (Exposed + existing imports). No new import needed.
